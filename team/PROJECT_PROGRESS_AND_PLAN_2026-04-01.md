# ClawOS Team 项目版本进度快照（2026-04-01）

> 本文用于“检查 Team 项目的版本进度”，仅聚焦**用户功能**与可见交付，不包含审计、运营、KPI 等非用户侧需求。

## 1) 当前版本判断

- **版本阶段**：`Team v1` 已进入可联调/可验收阶段（非仅原型）。
- **当前主入口**：`/app`，并兼容旧 `/console/*` 到 `/app/*` 的跳转。

## 2) M1（目标日期：2026-04-05）进度细化

结论：**M1 用户功能主链路已具备，当前约 85%~90%**。

### 2.1 已完成（用户可直接感知）

1. **注册/登录/创建公司流程可用**
   - 支持注册、登录、登出与创建公司。
2. **Admin 配置闭环可用**
   - 公司品牌配置。
   - Gateway 测试与 Agent 同步。
   - Team 创建与 Invite 发放。
3. **Invite 入会与会话闭环可用**
   - 用户可通过 invite token + nickname 加入。
   - 可创建并持续访问会话。
4. **消息能力可用**
   - 文本消息。
   - 图片/文件附件。
   - Agent 回复流。
5. **旧入口兼容可用**
   - 老 `/console/login`、`/console/register`、`/console/companies/new` 已可跳转到 `/app` 新流程。

### 2.2 待补齐（M1 收口项，仅用户功能）

1. **Onboarding 文案统一**
   - `/app` 注册、登录、创建公司三段文案统一成 Team 口径。
2. **Admin 首屏引导优化**
   - 把“company -> gateway -> agents -> teams -> invites”做成可视化步骤提示。
3. **Invite 首次进入体验优化**
   - 失败提示（过期/无效/次数耗尽）统一成面向用户可理解的错误文案。
4. **聊天页空态与附件提示优化**
   - 首次进入无消息、上传中、上传失败的提示语统一。
5. **并入 web 站内 Team 专页 + 首页 Team section 的正式对外文案与转化链路集中交付**
   - Team 专页：用户价值、核心能力、典型场景、CTA（立即体验）。
   - 首页 Team section：价值摘要 + 进入 Team 专页的单一主 CTA。
   - 转化链路：`首页 Team section -> Team 专页 -> /app`。

## 3) Team v1 用户功能清单（当前可演示）

- 公司级品牌设置、Gateway 连接测试、主 Agent 同步。
- Team 与 Invite 管理（有效期、次数上限）。
- 邀请入会（nickname）与成员会话持久化。
- 文本消息、图片/文件附件、Agent 回复流。
- `/app` 统一入口与旧路由兼容跳转。

## 4) 本周建议（2026-04-01 ~ 2026-04-07，仅用户功能）

1. **冻结范围**：只做 M1 用户功能收口，不新增非用户功能。
2. **优先级 P0**：先交付 onboarding + invite 失败提示 + 聊天空态文案统一。
3. **优先级 P1**：完成 web Team 专页、首页 Team section 与三段式转化链路。
4. **验收方式**：按“注册登录 -> 创建公司 -> Admin 配置 -> 发邀请 -> 入会聊天 -> 首页转化跳转”全链路走查。

## 5) 发布口径（仅用户功能）

- **一句话**：Team v1 已可完成从管理员配置到成员邀请入会与聊天协作的完整用户链路。
- **对外承诺边界**：本阶段只承诺用户可见功能体验，不扩展到运营后台与审计能力。
