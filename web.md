# Web 首页内容重构文档（V2）

> 按照最新要求，首页内容统一收敛为 5 个主模块：
> 1）主机业务 → 2）ClawOS 软件 → 3）定制扩展 OpenClaw → 4）集群管理/无人公司搭建 → 5）OEM/ODM 合作。

---

## 一、整体叙事主线

首页建议采用“**硬件入口 → 软件能力 → 扩展能力 → 规模化运营 → 深度合作**”的线性叙事：

1. 先讲清楚我们卖什么（主机业务）；
2. 再讲软件核心价值（ClawOS）；
3. 再讲可扩展能力（OpenClaw 定制）；
4. 再讲规模化场景（集群与无人公司）；
5. 最后承接高客单合作（OEM/ODM 软硬一体定制）。

这样可以兼顾三类用户：
- 直接采购用户（关注“买什么、能不能用”）；
- 技术决策用户（关注“能否扩展、是否可控”）；
- 企业合作用户（关注“是否能联合定义产品与交付”）。

## 二、首页结构（按模块重排）

## 1）主机业务（硬件入口）
**模块目标：先建立“可交付产品”的信任基础。**

建议呈现内容：
- 主机产品定位：面向企业 AI 业务的边缘计算/执行节点。
- 产品线建议（按“学习入门 → 行业落地”分层）：
  1. **通用 OpenClaw 主机**（入门学习，自主设置）
     - 适合对象：个人开发者、技术团队 PoC、内部试验环境；
     - 核心特点：提供通用基础环境，支持用户按需安装技能、配置流程和角色；
     - 价值点：低门槛启动，便于从 0 到 1 熟悉 OpenClaw 体系。
  2. **行业主机**（预设技能，开箱即用）
     - 适合对象：希望快速落地业务场景的团队；
     - 核心特点：内置行业模板与预设技能，减少前期配置成本；
     - 价值点：从“能跑”直接进入“可用、可运营”。
- 行业主机可细分方向：
  - 内容生成主机；
  - 客户管理主机；
  - 行业专家主机。
- 通用卖点（两类主机共性）：
  - 稳定运行（长期在线任务调度）；
  - 云边协同（与 ClawOS 控制台联动）；
  - 可扩展（可继续接入 OpenClaw 定制能力）。
- CTA：`查看主机型号对比` / `申请行业主机演示`。

文案示例：
- 标题：`主机业务：从入门学习到行业开箱`
- 副标题：`通用 OpenClaw 主机支持自主设置，行业主机内置预设技能，帮助团队快速完成业务落地。`

---

## 2）ClawOS 软件（核心控制平面）
**模块目标：说明软件是全局“大脑”，不是单点工具。**

建议呈现内容：
- ClawOS 定位：统一控制平面，负责目标、任务、流程、监控、告警。
- 核心能力（建议按“管控闭环”表达）：
  1. 目标拆解与任务编排；
  2. 多角色 AI 员工协同；
  3. 执行状态可视化与审计追踪。
- 用户收益：减少工具切换、降低协作损耗、提升执行可预测性。
- CTA：`下载体验` / `查看功能清单`。

文案示例：
- 标题：`ClawOS 软件平台`
- 副标题：`让企业从“工具堆叠”升级为“统一调度与治理”的 AI 运营系统。`

---

## 3）定制扩展 OpenClaw（能力扩展层）
**模块目标：突出“可按行业与场景扩展”，建立技术可塑性认知。**

建议呈现内容：
- OpenClaw 定位：围绕业务流程进行模块化扩展与能力接入。
- 扩展方向：
  - MCP 能力接入（营销、投研、IT、运维等）；
  - 企业系统集成（内部系统/API/数据源）；
  - 流程与角色模板定制（按组织职责配置）。
- 输出形式：插件、流程模板、行业解决方案包。
- CTA：`查看扩展能力` / `咨询定制开发`。

文案示例：
- 标题：`OpenClaw 定制扩展`
- 副标题：`将通用能力快速转化为你的行业工作流与业务资产。`

---

## 4）集群管理 / 无人公司搭建（规模化运营层）
**模块目标：回答“单点可用之后，如何规模化复制”。**

建议呈现内容：
- 集群管理：多节点统一纳管、任务分发、状态巡检、异常告警。
- 无人公司搭建：
  - 组织层：Owner/管理层策略控制；
  - 业务层：投资/营销/IT 等业务单元；
  - 执行层：AI 员工与流程自动化执行。
- 关键价值：从“项目化试点”走向“公司级运营”。
- CTA：`查看集群方案` / `获取无人公司搭建指南`。

文案示例：
- 标题：`集群管理与无人公司搭建`
- 副标题：`把 AI 能力从一台机器扩展到可治理、可审计、可复制的组织系统。`

---

## 5）OEM/ODM 合作（深度合作层）
**模块目标：承接高阶客户，明确联合定义与交付能力。**

建议呈现内容：
- 合作类型：
  - OEM：品牌与产品形态定制；
  - ODM：软硬件联合定义与研发交付。
- 定制范围：
  - 软件：界面、流程、角色、权限、MCP 组合；
  - 硬件：外观、规格、预装系统、启动流程；
  - 交付：部署、培训、运维、持续升级。
- 合作流程（建议 4 步）：需求评估 → 方案确认 → 打样部署 → 批量交付。
- CTA：`提交合作需求` / `预约方案沟通`。

文案示例：
- 标题：`OEM/ODM 合作`
- 副标题：`提供软件硬件深度定制方案，支持从 PoC 到规模化商用落地。`
