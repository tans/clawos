# 竞品站点（Moltybox）信息架构分析与我方改造建议（基于 /、/hardware、/enterprise）

> 说明：Moltybox 为竞品名称。建议中涉及“品牌名”均指我方应使用后台可配置品牌字段，不在前端文案中写死。

## 一、当前观察（竞品信息层面）

1. **三页定位重叠明显**
   - `/`、`/hardware`、`/enterprise` 都重复强调模型数、技能数、Agent 数，导致入口区分度不足。
   - `/hardware` 与 `/enterprise` 同时覆盖本地优先、安全、可观测、协作等主题，页面职责边界不清晰。

2. **术语体系混用，认知负担高**
   - 品牌名、平台名、产品名、组件名交替出现，且中英文混排较多。
   - 首访用户难以快速理解“品牌—产品—能力”的层级关系。

3. **CTA 过多且并列，决策路径不线性**
   - 同层出现“立即开始/联系专家/下载客户端/查看文档/进入某功能”等多个动作。
   - 缺少按角色（开发者、业务负责人、采购）分流后的主行动路径。

4. **决策信息不足**
   - 能力点很多，但缺少标准化选型结构：
     - 何时优先选择 Hardware？
     - 何时进入 Enterprise 方案？
     - 本地部署与企业托管在成本、时效、治理上的取舍规则是什么？

---

## 二、我方改造目标

1. 首访 30 秒完成产品分流：用户知道自己应该进入 Hardware、Enterprise、Marketplace 或 Store。
2. 技术负责人 3 分钟完成可行性判断：架构、安全、集成、运维是否匹配。
3. 业务与采购 5 分钟完成商业判断：实施周期、ROI、服务边界是否可接受。

### 执行优先级（重要）

**首先改造 Web 首页（`/`）**，再逐步推进 `/hardware`、`/enterprise`、`/marketplace`、`/store`。  
原因：首页承担全部首访流量分发，先优化首页可最快提升分流效率与线索质量。

---

## 三、建议的信息架构（加入你方产品特色）

建议采用“1 个总入口 + 4 条核心产品线 + 2 条辅助转化线”：

- **总入口**：`/`（价值总览 + 用户分流）
- **产品线**：`/hardware`、`/enterprise`、`/marketplace`、`/store`
- **辅助线**：`/solutions`（场景方案）、`/security`（安全治理）

### 1) 首页 `/`：只做导航与分流

首页仅承担三件事：
1. 一句话讲清核心价值（解决什么业务问题）。
2. 四张产品卡清晰分流（Hardware / Enterprise / Agent 众包市场 / 商城）。
3. 用证据（案例、数据、客户）增强信任并引导 CTA。

推荐模块顺序：
1. Hero（主 CTA：预约演示；次 CTA：联系顾问）
2. 产品分流四卡（适合人群 + 典型场景 + 上线周期）
3. 业务结果指标（效率、稳定性、交付时长）
4. 客户案例摘要
5. Agent 众包市场精选任务与优秀创作者榜（跳转 `/marketplace`）
6. 商城热销模板/硬件套餐（跳转 `/store`）
7. 安全治理摘要（跳转 `/security`）
8. FAQ（5 个关键问题）

### 2) `/hardware`：强调本地执行与边缘自动化

页面只回答四个问题：
1. 为什么需要硬件（隐私、稳定性、设备控制）
2. 能力边界是什么（桌面自动化、工作流执行、外设/IoT 接入）
3. 接入复杂度如何（部署步骤、依赖要求、上线时间）
4. 商务路径如何走（规格、交付、试点、售后）

建议新增：
- 规格对比表（算力、内存、并发建议、适用团队）
- POC 流程图（申请→交付→上线→验收）
- 与 Enterprise 的边界说明（避免重复售卖）

### 3) `/enterprise`：强调组织级治理与规模化交付

聚焦四类信息：
- 治理：权限、审计、策略、回滚
- 集成：SSO、VPC、内网系统、数据系统
- 运维：可观测、成本分析、SLA
- 交付：试点范围、里程碑、ROI 模板

推荐模块顺序：
1. 从 Demo 到生产的企业价值
2. 企业能力矩阵（治理/安全/协作/运维/集成）
3. 参考架构（本地/混合）
4. 行业场景方案
5. 试点方法论
6. 销售咨询 CTA

### 4) `/marketplace`：Agent 众包市场（新增）

定位：连接“需求方（企业/团队）”与“供给方（Agent 开发者/工作流设计者）”。

建议模块顺序：
1. 市场价值说明（为什么用众包市场）
2. 任务大厅（按行业、预算、交付周期筛选）
3. 创作者能力画像（评分、完成率、行业经验、响应时效）
4. 交易与托管机制（里程碑、验收、仲裁）
5. 企业发布任务 CTA + 创作者入驻 CTA

建议补充机制说明：
- 任务模板：客服自动化、线索挖掘、内容生产、数据分析
- 质量治理：交付标准、评分权重、违规处理
- 数据安全：脱敏协作、权限隔离、审计留痕

### 5) `/store`：商城（新增）

定位：沉淀可直接购买/订阅的产品化资产（硬件、Agent 模板、行业包、服务包）。

建议模块顺序：
1. 商城首页导航（硬件 / 模板 / 行业包 / 服务包）
2. 商品卡片（能力说明、适用场景、价格、交付方式）
3. 对比区（不同套餐功能差异）
4. 购买流程与售后说明（发票、退款、升级、交付 SLA）
5. 立即购买 CTA + 咨询方案 CTA

建议新增商品分类：
- 硬件类：不同规格设备与配件
- 数字资产类：Agent 模板、工作流脚本、行业知识包
- 服务类：部署服务、代运营服务、专家咨询

## 四、命名与文案治理建议（按你方新增要求修正）

1. **品牌名不写死**
   - 前端展示名统一读取后台配置（例如 `brand_display_name`），避免硬编码品牌。

2. **统一命名层级**
   - 品牌（后台配置）
   - 平台（如 Agent Platform）
   - 产品（Hardware / Enterprise / Marketplace / Store）
   - 组件（Runtime / Memory / Skills / Agent OS）

3. **每页坚持单一主叙事**
   - 首页：分流
   - Hardware：本地执行
   - Enterprise：组织治理

---

## 五、转化路径改造（高优先级）

1. 角色化 CTA：
   - 开发/技术：查看文档、申请技术评估
   - 业务负责人：预约演示
   - 采购/管理层：联系方案顾问

2. 每页保持“1 主 CTA + 1 次 CTA”，避免按钮并列竞争。

3. 增加“选型向导”（3 问）：
   - 数据敏感度
   - 组织规模
   - 上线时效
   输出推荐：Hardware / Enterprise / Marketplace / Store。

4. 在首页新增双漏斗 CTA：
   - 需求侧：发布任务（Marketplace）/ 购买方案（Store）
   - 供给侧：创作者入驻（Marketplace）/ 上架商品（Store）

## 六、SEO 与可发现性建议

1. `/`、`/hardware`、`/enterprise`、`/marketplace`、`/store` 使用独立 Title/Description，避免复用同一元信息。
2. 为 `/hardware`、`/enterprise`、`/marketplace`、`/store` 添加结构化数据（FAQ、Product、Organization、Offer）。
3. 新增方案页：
   - `/solutions/operations-automation`
   - `/solutions/enterprise-agent-governance`
   - `/solutions/local-first-ai-execution`

## 七、4 周落地路线图

### 第 1 周：先改首页（最高优先级）
- 完成首页 Hero、四产品分流卡、双漏斗 CTA 的首版上线
- 完成首页信息去重：仅保留“价值主张 + 分流 + 证据 + 行动”
- 埋点首页关键事件：卡片点击、CTA 点击、滚动深度、留资入口点击

### 第 2 周：页面重排与去重
- 输出术语字典与命名规范（含后台品牌字段规则）
- 定义五页“受众—主信息—主 CTA”（`/`、`/hardware`、`/enterprise`、`/marketplace`、`/store`）
- 首页改为四产品分流
- Hardware 与 Enterprise 做边界化改写
- Marketplace 与 Store 完成首版信息框架

### 第 3 周：转化组件上线
- 角色化 CTA
- 选型向导 + 案例卡片 + 市场任务流转组件

### 第 4 周：数据验证与迭代
- 监测：首页分流点击率、留资率、有效线索率、任务发布率、商城下单率
- A/B 测试 Hero 与分流卡文案
- 根据首页数据回流，迭代二级页首屏文案与 CTA 优先级

### 首页先行改造的验收标准（DoD）

1. 首页首屏在 5 秒内可理解“我该点哪里”（四卡分流清晰）。
2. 首页仅保留 1 主 CTA + 1 次 CTA（避免并列抢点击）。
3. 首页到四条产品线点击率较改造前提升（建议目标：+20% 起）。
4. 首页线索入口点击率提升且无效线索占比不升高。

---

## 八、核心指标

- 首页到 `/hardware`、`/enterprise`、`/marketplace`、`/store` 的分流点击率
- 产品页到留资页转化率
- 有效线索占比（可被销售跟进）
- Marketplace 任务发布转化率、任务完结率、创作者复购率
- Store 下单转化率、客单价、复购率
- 首次会话回退率（信息清晰度）
